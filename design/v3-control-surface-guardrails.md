# V3 控制面收敛与布局 Guardrails

状态：下一阶段设计入口，待确认后实现。

## 背景

V3 已经具备灵活的 Region / Slot / Content 架构。当前问题不是能力不足，而是底层自由度过多暴露给普通用户，导致水印元素容易重叠、越界或视觉失控。

产品方向应从“开放编辑器”收敛为“模板驱动 + 安全微调”。

## 目标

1. 普通用户能快速得到稳定、好看的水印结果。
2. 高级用户仍可进入结构编辑，但默认不打扰普通流程。
3. 系统能提示或阻止明显的重叠、越界和无意义组合。
4. 预设成为安全模板，而不是随意可破坏的配置样例。

## 非目标

- 暂不实现完整 Figma 式自由编辑器。
- 暂不实现拖拽所有元素。
- 暂不实现模板市场。
- 暂不做账号系统或云端同步。

## 控制面分层

### 主界面暴露

面向普通用户，低风险、高频：

- 图片上传与当前图片选择。
- 水印预设选择。
- 内容开关：相机、镜头、参数、日期、Logo、签名、自定义文本。
- 全局自定义文本。
- 整体大小：小 / 中 / 大。
- 颜色模式：黑 / 白 / 暖灰 / 自动。
- Logo 大小：小 / 中 / 大。
- 底栏密度：紧凑 / 标准 / 宽松。
- 一键恢复当前预设。

### 高级设置暴露

面向懂结构的用户，默认折叠：

- Region 启用 / 禁用。
- Slot 启用 / 禁用。
- Slot 内容字段顺序。
- Slot 级字号、颜色、粗细。
- separator。
- 底栏 margin / 高度。
- side-edge 宽度。
- free region anchor / offset。

### 暂时隐藏

当前收益低、风险高：

- 任意新增 Region。
- 任意删除核心 Region。
- 任意 slot id。
- 直接编辑 Region id。
- pixel / ratio 单位自由切换。
- 直接编辑 anchor 字符串。
- 直接编辑 font family 文件名。
- line-height 和 size_reference 的细粒度控制。
- 所有元素自由拖拽。

## Guardrails

### 必须检测

- 元素矩形重叠。
- 元素超出画布。
- 元素覆盖图片主体但预设不允许。
- 文字为空但 slot 启用。
- Logo / 签名资源缺失。
- 底栏高度不足以容纳当前字号。

### 建议响应

- 主界面：给出“布局可能重叠”的轻提示，提供一键恢复。
- 高级界面：标红冲突 slot，显示冲突对象。
- 严重越界：阻止正式 Process，允许继续编辑。

## 实施顺序

1. 盘点当前 UI 所有控件，标记主界面 / 高级 / 隐藏。
2. 移除主界面的新增 Region / 自由定位入口。
3. 增加预设参数化控制：大小、颜色、密度、内容开关。
4. 增加 layout diagnostic：输出 overlap / out-of-bounds warnings。
5. 在 UI 展示 warnings，并加入恢复预设。
6. 再评估拖拽、保存预设和撤销/重做。

## 验收标准

- 默认预设打开后无重叠 warning。
- 普通用户不进入高级设置，也能完成上传、预设、预览、处理和下载。
- 主界面无法创建任意 Region 或任意 Slot。
- 高级设置能看到结构，但危险项默认折叠或隐藏。
- 人为制造重叠时，UI 能指出冲突。
- 全量测试和前端 build 通过。
- 线上部署后完成截图检查。
